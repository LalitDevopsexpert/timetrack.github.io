What's new if compare with aTimeLogger

Web account is required

In aTimeLogger web sync was optional while it’s required in new app.
Why:
  • it’s easier in development;
  • your data is always backed up
It doesn’t mean you always need internet connection - it syncs when possible.

Tabs changed to Left Menu

Now you have more space for data. If lot of people will ask for tabs we’ll add it as an option

Tags

Tagging allows you to track more and get more detailed reports. You can add multiple tags to activities. Tags are also supported in goals and reports

Goals

  • more notifications options. You can set what time before OR after goal you’ll get notification;
  • activate/deactivate goals. In previous version goals were always activated. But now you can create goal like ‘Study 7 hours a day’ and activate it only during exams
  • 'number of occurrences' goals. You can create now something like ‘Drink not more than 2 cups of coffee a day'
  • ‘total duration’ goal;
  • as previously mentioned you can now add not only types but also tags to goals;

Reports

  • pie charts and bar charts;
  • save report templates

Push notifications

Push notifications solve 2 problems:
  • If you have iPhone and iPad and do something on iPhone then your changes appear immediately on iPad
  • Today Extension cannot access local notifications that are used in goal notifications. It means that if you set goal notification and use Today Extension to start/stop/pause/resume local notification won’t appear. The only solution I see now is to use remote push notifications
Also we’ll be able to send smart suggestions (e.g you changed location but didn’t change activity) in future

Geo (will appear later)

  • you’ll see where you spend your time;
  • with geo information collected we can create automatic switch between activities in different geo locations;

Calendar (will appear later)

  • scheduling, recurring activities, etc.

Projects and Rates (will appear later)

  • many freelancers use aTimeLogger. It can be good addition for them.

Integrations with devices (will appear later)

  • apple watch, pebble, etc.
